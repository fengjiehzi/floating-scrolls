## 一、项目简介

「万卷浮生」是一款融合古典名著阅读、AI 互动剧情、角色养成与跨作品 PVP 对战的 Web 游戏。玩家可导入古典名著（西游记、三国演义、水浒传、红楼梦、封神演义、白蛇传等），由 AI 生成互动剧情章节，在阅读中收集角色卡与法宝，最终让不同作品中的角色同台竞技——孙悟空对战奥特曼、关羽对决哪吒，皆可实现。

### 核心特色

- 📖 **典籍阅读 + AI 剧情**：12 部预置名著，AI 根据原文生成分章节互动剧情，玩家选择影响故事走向
- 🎴 **24 个角色卡**：覆盖西游记、三国演义、水浒传、红楼梦、封神演义、白蛇传、山海经、史记、奥特曼 9 大来源
- 🏺 **31 件法宝**：金箍棒、青龙偃月刀、乾坤圈、风火轮等传奇道具
- ⚔️ **跨作品 PVP 对战**：WebSocket 实时对战，支持跨维度压制（神话 vs 凡人）、等级压制（S/A/B）、秒杀机制
- 🤖 **11 家国产大模型**：DeepSeek、智谱、Kimi、通义、MiniMax、豆包、阶跃、硅基流动、Mimo、OpenAI、Ollama，全部支持 OpenAI 兼容 + Anthropic 原生双协议
- 🎨 **玻璃拟态 UI**：动态背景、毛玻璃效果、流畅动画，移动端自适应

## 三、快速开始

### 3.1 安装与启动

```bash
# 1. 进入项目目录
cd 万卷浮生_Web应用

# 2. 安装依赖
npm install

# 3. 启动服务器（前台运行）
npm start
# 或：node server.js
```

**启动方式说明**：

| 方式 | 命令 | 适用场景 |
|------|------|----------|
| 前台运行（推荐） | `npm start` 或 `node server.js` | 日常开发调试，可看到实时日志，`Ctrl+C` 即可关闭 |
| 后台运行（Windows） | `start /B node server.js` | 需在后台保持运行，关闭终端后服务不中断 |
| 后台运行（macOS/Linux） | `nohup node server.js > server.log 2>&1 &` | 同上，日志输出到 `server.log` |
| 开发模式（热重载） | `npx nodemon server.js` | 二次开发时修改代码自动重启 |

启动成功后终端会显示：
```
[DB] 已初始化 23 个预置角色
[DB] 已初始化 31 件预置道具
[DB] 已初始化 8 个机器人玩家
[服务器] 运行在 http://localhost:3000
```

> 💡 默认端口 3000，可在 `.env` 文件中通过 `PORT=端口号` 修改。

### 3.2 关闭服务器

#### 方式一：前台运行时关闭（最常用）

在运行 `npm start` 或 `node server.js` 的终端窗口中，按组合键：

```
Ctrl + C
```

终端会显示 `已终止` 或退出命令提示符，服务器随即停止。

#### 方式二：通过端口查找并终止（适用于后台运行或终端已关闭）

**Windows（PowerShell / CMD）**：

```bash
# 1. 查找占用 3000 端口的进程 PID
netstat -ano | findstr :3000
# 输出示例：TCP 0.0.0.0:3000 0.0.0.0:0 LISTENING 12345
#                                                        ↑ 最后一列为 PID

# 2. 终止该进程（将 12345 替换为实际 PID）
taskkill /PID 12345 /F
```

**macOS / Linux**：

```bash
# 方式 A：通过 lsof 查找 PID 后终止
lsof -i:3000
kill -9 <PID>

# 方式 B：一行命令搞定
kill -9 $(lsof -t -i:3000)

# 方式 C：按进程名终止
pkill -f "node server.js"
```

#### 方式三：通过任务管理器关闭（仅 Windows）

1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 切换到「详细信息」标签页
3. 按名称排序找到 `node.exe` 进程
4. 右键 →「结束任务」

#### 方式四：修改端口避让冲突

如果不想关闭现有进程，可修改本应用端口：

```bash
# 编辑 .env 文件
PORT=3001
# 重启服务器即可在 3001 端口运行
```

#### 关闭验证

- 浏览器访问 `http://localhost:3000` 显示「无法访问此网站」即表示已关闭
- Windows 可执行 `netstat -ano | findstr :3000`，无输出即端口已释放

### 3.3 访问应用

打开浏览器访问：**http://localhost:3000**

### 3.4 首次使用流程

1. 在欢迎页点击「开启浮生之旅」
2. 注册账号（用户名 + 密码）
3. 登录后进入主界面
4. 进入「🤖 AI 设置」配置你的 AI 服务商与 API Key
5. 回到「📚 典籍库」选择名著开始剧情，或在「⚔️ 对战大厅」直接开战

## 四、AI 配置（重要）

AI 是本作的核心驱动，需在「🤖 AI 设置」页面配置后才能体验完整功能（AI 剧情、AI 角色生成、AI 解说）。

### 4.1 支持的 11 家服务商

| 服务商 | 默认模型 | 支持 Anthropic 原生协议 |
|--------|----------|------------------------|
| DeepSeek 深度求索 | deepseek-v4-pro | ✅ |
| 智谱 AI (GLM) | glm-5.2 | ✅ |
| 月之暗面 Kimi | kimi-k2.6 | ❌ |
| 通义千问 (Qwen) | qwen3.6-plus | ✅ |
| MiniMax 稀宇 | abab6.5s-chat | ✅ |
| 豆包 (Doubao) | doubao-pro-256k | ✅ |
| 阶跃星辰 StepFun | step-2-16k | ✅ |
| 硅基流动 SiliconFlow | Qwen2.5-72B-Instruct | ✅ |
| 小米 MiMo | mimo-7b-rl | ✅ |
| OpenAI 官方 | gpt-4o | ✅ |
| 本地 Ollama | llama3.1 | ❌ |
| **自定义供应商** | 用户填写 | 取决于接口 |

### 4.2 双协议支持

每个服务商（除 Kimi、Ollama 外）均同时支持两种调用协议，可在设置页切换：

- **OpenAI 兼容协议**：使用 `/chat/completions` 端点，`Authorization: Bearer <key>` 头
- **Anthropic 原生协议**：使用 `/v1/messages` 端点，`x-api-key` 头，system 字段置于顶层

切换协议后，接口地址（Base URL）会自动适配。若使用官方默认地址，Base URL 留空即可。

### 4.3 自定义模型

选择「自定义供应商」后，需手动填写：

- **模型名称**：如 `my-model-v1`
- **接口地址**：OpenAI 格式（如 `https://api.example.com/v1`）或 Anthropic 格式（如 `https://api.example.com/anthropic`）
- **API Key**：你的密钥

适用于自建模型服务、第三方代理、未内置的模型厂商。

### 4.4 配置步骤

1. 进入「🤖 AI 设置」页
2. 选择服务商（下拉列表）
3. 选择模型（下拉列表，或自定义供应商时手填）
4. 选择协议（OpenAI / Anthropic，仅支持的才显示）
5. 填写 API Key（已保存的留空则不修改）
6. 可选：填写自定义接口地址
7. 点击「保存配置」
8. 可选：点击「测试连接」验证配置是否生效

> 💡 配置保存在 `user_ai_config` 数据库表，API Key 加密存储，按用户隔离。

## 五、游戏玩法

应用采用侧边栏导航，共 7 大功能模块：

### 5.1 📚 典籍库

- 浏览 12 部预置名著（聊斋志异、山海经、搜神记、太平广记、子不语、阅微草堂笔记、西游记、三国演义、水浒传、红楼梦、封神演义、白蛇传）

- 浏览 24 个角色卡，按作品来源分组

- 角色分级：S（神话级）> A（英雄级）> B（凡人级）

| 西游记（3） | 孙悟空、唐僧、猪八戒 |

  - 秒杀机制：单次伤害超过目标最大生命 60% 时触发「⚡ 秒杀！」

## 八、常见问题（FAQ）

### Q1：启动时提示「端口 3000 被占用」

```bash
# 查找占用进程
netstat -ano | findstr :3000
# 终止进程（替换 PID）
taskkill /PID <PID> /F
# 或修改 .env 中的 PORT 为其他端口
```

### Q2：点击「保存配置」后一直显示「保存中」

**原因**：浏览器缓存了旧版 JS 文件。

**解决**：按 `Ctrl + F5` 强制刷新一次。新版已添加 `Cache-Control: no-cache` 头与 15 秒超时保护，后续不会再出现此问题。

### Q3：AI 连接测试失败

- 检查 API Key 是否正确
- 检查所选协议是否与你的接口地址匹配
- 网络问题：部分服务商需确保能访问其 API 域名
- 查看终端是否有错误日志

### Q4：角色卡库显示不全

服务器启动时会自动补录缺失角色（按名称比对，避免覆盖自定义角色）。若仍不全：

```bash
# 重启服务器即可
npm start
```

### Q5：自定义角色 id 冲突

系统已处理：当预置角色 id 被自定义角色占用时，会自动分配新 id，不会覆盖你的自定义角色。

### Q6：如何添加新角色

1. 编辑 `characters-data.js`，按现有格式追加角色对象
2. 将角色图片放入 `public/images/classics/characters/`（PNG 格式，kebab-case 命名）
3. 重启服务器，数据库自动补录

### Q7：如何添加新书籍

1. 编辑 `db.js` 中 `classicBooks` 数组，追加书籍对象
2. 重启服务器，数据库按标题比对自动补录

### Q8：移动端能否使用

可以。前端采用响应式布局，适配手机屏幕。移动端底部有导航栏，功能与桌面端一致。

### Q9：如何与好友对战

1. 双方各自注册账号并登录
2. 一方进入「对战大厅」→「在线匹配」
3. 另一方同样操作，系统会自动匹配
4. 也可通过 WebSocket 邀请指定好友

### Q10：机器人对战会记录战绩吗

会。机器人对战结果同样写入 `battles` 表，影响排行榜。

## 九、开发指南

### 添加新 AI 服务商

1. 在 `ai-model-presets.js` 中添加新服务商配置（含 `openai_base_url`、`anthropic_base_url`、`models` 数组）
2. 在 `.env.example` 中添加对应的环境变量
3. 在 `server.js` 的 AI 端点中处理新服务商的 API Key 读取
4. 重启服务器，前端「AI 设置」页会自动显示新服务商

### 修改战斗规则

编辑 `battle-engine.js`，关键参数：

- `MAX_ROUNDS`：最大回合数（当前 12）
- 等级压制倍率：S 对 A 为 1.2x，S 对 B 为 1.5x
- 秒杀阈值：单次伤害 > 目标最大生命 60%
- 跨维度压制：神话级对凡人级额外 1.3x

### 数据库迁移

数据库使用 better-sqlite3（同步 API），表结构在 `db.js` 的 `initDB()` 中定义。新增列时使用 `ALTER TABLE ... ADD COLUMN` 并 try-catch 忽略「列已存在」错误，实现兼容升级。
