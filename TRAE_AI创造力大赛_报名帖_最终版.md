# 【生活娱乐赛道】万卷浮生 —— 把任意小说角色变成可对战卡牌

**标签**：#生活娱乐

---

## 一、创意名称 + 创意介绍

**创意名称**：万卷浮生

**想解决什么问题**：
每个读者心里都有一个永恒的疑问——"孙悟空和迪迦谁更厉害？""诸葛亮和梅林谁更聪明？""林黛玉和薛宝钗谁魅力更高？"这类跨作品战力讨论在B站、贴吧、知乎常年热门，但永远没有答案，因为不同作品的设定无法直接对比。读者只能嘴上吵，没法"打一架"见分晓。

**为什么会想到做这个**：
B站"孙悟空vsXXX"战力对比视频动辄百万播放，评论区吵翻天，但永远没有结论——因为没有一个平台能让这些角色真正"打一场"。现有的卡牌对战游戏角色都是预设的，玩家无法把自己喜欢的小说角色变成卡牌。而AI已经能理解长文本、提取角色特征、量化属性数值——技术上已经具备条件。

**大概是什么产品**：
一个Web端的AI角色卡牌PVP游戏——玩家上传公版小说文本，AI自动提取角色并科学计算战力属性，生成可对战的角色卡牌。玩家通过剧情养成角色，然后选一个角色与其他玩家的角色进行1v1自动战斗PVP，用战斗结果回答"谁更厉害"。

---

## 二、目标用户及痛点

**面向哪些用户**：
- 战力讨论爱好者（核心用户）：活跃在B站、贴吧、知乎的"XXvsYY"话题下，渴望"真打一场"见分晓
- 卡牌对战游戏玩家：喜欢《阴阳师》《Fate/GO》，但苦于想要的角色官方不出
- 小说/动漫重度粉丝：对某些角色有强烈情感，想"拥有"这个角色并培养他变强
- 直播/短视频内容创作者："我用AI让孙悟空和迪迦打了一场"自带传播潜力

**在什么场景下使用**：
- 和朋友吵"孙悟空vs迪迦谁厉害"吵不出结果 → 各自上传小说养成角色，PVP见分晓
- 喜欢某个小说角色，官方游戏里没有 → 自己上传小说，AI生成角色卡牌开始养成
- 直播时需要互动内容 → 观众投票选择出战角色，AI自动战斗，弹幕押注

**当前痛点**：
| 痛点 | 现状 |
| --- | --- |
| 跨作品战力讨论无解 | 只能嘴上吵，永远没结论 |
| 卡牌游戏角色受限 | 官方出什么玩家玩什么 |
| 小说角色无法"拥有" | 读完小说只能记住，无法养成 |
| 战力对比无科学依据 | 各作品设定不同，无法直接对比 |

---

## 三、价值与意义

**商业价值**：
- 内容供给无限化：公版小说每本可提取数十个角色，角色卡牌供给近乎无限
- 天然社交传播："孙悟空vs迪迦"类话题自带百万级流量，获客成本极低
- PVP高粘性：《阴阳师》运营8年仍盈利，玩家为"让本命角色变强"持续投入

**社会价值**：
- 降低卡牌游戏创作门槛：任何玩家几分钟就能"造"出一批角色卡牌，真正实现"人人都是创造者"——契合TRAE大赛"世界很大，放手去造"的理念
- 经典文学的新生：《红楼梦》《西游记》等经典以"收集对战"方式触达新一代读者，玩家为养强孙悟空会主动去读原著
