### 1.3 架构关键设计

#### 设计 1：HTTP + WebSocket 共享端口
- **位置**：[server.js](file:///e:/万卷浮生/万卷浮生_Web应用/server.js#L518-L521)
- **实现**：`http.createServer(app)` 创建 HTTP 服务器，再通过 `WebSocketServer({ server })` 挂载 WebSocket
- **好处**：部署只暴露一个端口（3000），简化反向代理与 CORS 配置

#### 设计 2：战斗引擎纯函数化
- **位置**：[battle-engine.js](file:///e:/万卷浮生/万卷浮生_Web应用/battle-engine.js)
- **实现**：`executeBattle(char1, char2)` 接收纯数据，返回完整战斗状态，不读写 DB
- **好处**：易测试、易回放、易被未来桌面版（Godot 4）复用核心算法

#### 设计 3：机器人系统
- **位置**：[db.js](file:///e:/万卷浮生/万卷浮生_Web应用/db.js#L132-L161) + [ws-server.js](file:///e:/万卷浮生/万卷浮生_Web应用/ws-server.js#L27-L37)
- **实现**：8 个 `bot_*` 用户入库，随机战绩，启动时加载到 `botUsers` Map
- **好处**：玩家随时有对手，保证 PVP 体验不空转

#### 设计 4：WebSocket 状态机驱动
- **位置**：[ws-server.js](file:///e:/万卷浮生/万卷浮生_Web应用/ws-server.js)
- **实现**：服务端用 `onlineUsers` / `matchQueue` / `activeBattles` / `pendingInvites` 四个 Map 管理状态
- **好处**：单进程内有状态、调试简单、并发量足够（开源项目场景）

### 1.4 架构边界与限制

| 维度 | 当前能力 | 瓶颈 | 重构方向 |
|------|---------|------|----------|
| **并发用户** | 单进程数百 | 内存中 Map 多 | Godot 4 桌面端 / 多进程 |
| **战斗并发** | 同时多场（Map 索引） | 单 CPU | 无压力（计算很轻） |
| **AI 提取** | 纯规则（中文姓氏匹配） | 长文本 / 非中文 | 接入 LLM（双轮提取） |
| **数据库** | SQLite 单文件 | 多机部署 | 迁移至 PostgreSQL |
| **前端框架** | 原生 HTML/JS | 工程化弱 | 可迁移至 Vue/React |

### 2.2 关键依赖详解

#### better-sqlite3
- **为什么选**：相比 `sqlite3` 包，它是同步 API，性能高 5-10 倍
- **使用场景**：所有 DB 操作都同步，无 callback / Promise 嵌套
- **WAL 模式**：[db.js](file:///e:/万卷浮生/万卷浮生_Web应用/db.js#L15-L17) 通过 `db.pragma('journal_mode = WAL')` 启用，读写并发性能大幅提升
- **外键约束**：`db.pragma('foreign_keys = ON')` 显式开启（SQLite 默认关闭）

#### ws（而非 socket.io）
- **为什么选**：纯 WebSocket 协议，无 fallback 复杂度
- **消息格式**：统一 JSON 字符串，自描述
- **鉴权方式**：[URL query 传 token](file:///e:/万卷浮生/万卷浮生_Web应用/ws-server.js#L743-L760)，握手时一次性验证

#### bcryptjs（而非 bcrypt）
- **为什么选**：纯 JS 实现，无 native 编译，部署到任何平台都直接跑
- **salt rounds**：[auth.js](file:///e:/万卷浮生/万卷浮生_Web应用/auth.js#L30) 使用 10 轮（性能与安全平衡）

#### jsonwebtoken
- **签名算法**：HS256（对称密钥）
- **有效期**：[7 天](file:///e:/万卷浮生/万卷浮生_Web应用/auth.js#L8)
- **payload**：仅包含 `userId` 与 `username`

### 2.3 不使用的技术（明确排除）

| 技术 | 原因 |
|------|------|
| TypeScript | MVP 阶段不过度工程化，原生 JS 足够 |
| React/Vue | 同上，原生 DOM 操作可控 |
| Webpack/Vite | 同上，零构建步骤 |
| Redis | 单进程内存 Map 足够 |
| Docker | 部署目标用户为开发者，`node server.js` 即可 |
| PostgreSQL | SQLite 足够支撑开源项目规模 |

### 2.4 依赖管理规范

```json
// package.json 关键字段
{
  "name": "wanjuan-fusheng-web",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": {
    "start": "node server.js",
    "dev": "node server.js"
  }
}
```

**原则**：
- 所有依赖锁版本（避免 major 升级破坏）
- 不引入 `devDependencies`（保持简单）
- 升级依赖前必须跑通核心功能（造卡、对战、形态切换）

### 3.4 文件加载顺序

**后端启动顺序**（[server.js](file:///e:/万卷浮生/万卷浮生_Web应用/server.js#L22-L25)）：

```
1. require('./db')          → 触发 initDB()，自动建表 + 预置数据
2. require('./auth')        → 注入 JWT 工具
3. require('./ws-server')   → 注入 WebSocket 工具
4. const app = express()
5. app.use(中间件)           → cors, json, static
6. app.use(路由)             → 鉴权/角色/玩家数据/排行榜
7. http.createServer(app)
8. initWSServer(server)     → WebSocket 挂载 + 加载机器人
9. server.listen(PORT)
```

**前端加载顺序**（[index.html](file:///e:/万卷浮生/万卷浮生_Web应用/public/index.html)）：

```
1. <head> 引入 themes.css → style.css → animations.css
2. <body> 内联 <style> 提取 UI 样式
3. <script> 引入 game-toast.js → app.js → auth.js → characters.js → lobby.js → battle.js
4. DOMContentLoaded → App.init() → 鉴权检查 → 渲染当前页
```

### 6.2 关键函数

#### register([auth.js:11-48](file:///e:/万卷浮生/万卷浮生_Web应用/auth.js#L11-L48))

```javascript
function register(username, password, nickname, avatar) {
    // 1. 参数校验
    if (!username || !password) return {success:false, message:'用户名和密码不能为空'};
    if (username.length < 3 || username.length > 20) return {success:false, message:'用户名长度需3-20位'};
    if (password.length < 6) return {success:false, message:'密码长度至少6位'};

    // 2. 查重
    const existing = db.prepare('SELECT id FROM users WHERE username = ?').get(username);
    if (existing) return {success:false, message:'用户名已存在'};

    // 3. 加密
    const hashedPassword = bcrypt.hashSync(password, 10);

    // 4. 入库
    const result = db.prepare(`
        INSERT INTO users (username, password, nickname, avatar)
        VALUES (?, ?, ?, ?)
    `).run(username, hashedPassword, nickname || username, avatar || '🎮');

    // 5. 生成 Token
    const userId = result.lastInsertRowid;
    const user = db.prepare('SELECT id, username, nickname, avatar, wins, losses FROM users WHERE id = ?').get(userId);
    const token = jwt.sign({userId, username}, JWT_SECRET, {expiresIn: JWT_EXPIRES_IN});

    return {success:true, token, user};
}
```

#### login([auth.js:51-79](file:///e:/万卷浮生/万卷浮生_Web应用/auth.js#L51-L79))

```javascript
function login(username, password) {
    if (!username || !password) return {success:false, message:'用户名和密码不能为空'};

    const user = db.prepare('SELECT * FROM users WHERE username = ?').get(username);
    if (!user) return {success:false, message:'用户名或密码错误'};

    if (!bcrypt.compareSync(password, user.password)) {
        return {success:false, message:'用户名或密码错误'};
    }

    const token = jwt.sign({userId: user.id, username: user.username}, JWT_SECRET, {expiresIn: JWT_EXPIRES_IN});

    return {success:true, token, user: {
        id: user.id, username: user.username, nickname: user.nickname,
        avatar: user.avatar, wins: user.wins, losses: user.losses
    }};
}
```

#### authMiddleware([auth.js:88-105](file:///e:/万卷浮生/万卷浮生_Web应用/auth.js#L88-L105))

```javascript
function authMiddleware(req, res, next) {
    const authHeader = req.headers.authorization;
    if (!authHeader || !authHeader.startsWith('Bearer ')) {
        return res.status(401).json({error: '未提供认证token'});
    }
    const token = authHeader.split(' ')[1];
    try {
        const decoded = jwt.verify(token, JWT_SECRET);
        const user = getUserById(decoded.userId);
        if (!user) return res.status(401).json({error: '用户不存在'});
        req.user = user;
        next();
    } catch (err) {
        return res.status(401).json({error: 'token无效或已过期'});
    }
}
```

### 9.3 伤害计算（[battle-engine.js:73-121](file:///e:/万卷浮生/万卷浮生_Web应用/battle-engine.js#L73-L121)）

```javascript
function calculateDamage(attacker, defender, skill) {
    // 1. 基础伤害
    let base;
    if (skill.type === 'physical_attack') base = attacker.stats.power * 1.2;
    else if (skill.type === 'magic_attack') base = attacker.stats.special_ability * 1.2;
    else if (skill.type === 'speed_attack') base = attacker.stats.speed * 1.2;
    else if (skill.type === 'summon') base = attacker.stats.power * 0.6;
    else base = attacker.stats.power * 1.0;

    base += attacker.stats.special_ability * 0.3;

    // 2. 暴击
    const critRate = Math.min(0.35, (attacker.stats.speed + attacker.stats.intelligence) / 400);
    const isCrit = Math.random() < critRate;
    const critMult = isCrit ? 1.5 : 1.0;

    // 3. 防御减免
    const defReduction = 1 - (defender.stats.defense / (defender.stats.defense + 200));

    // 4. 等级压制
    const gradeMult = getGradeMult(attacker, defender);

    // 5. 次元压制
    const dimensionMult = getDimensionMult(attacker, defender);

    // 6. 攻击降低 debuff
    const attackDownMult = attacker.buffs.attack_down > 0 ? 0.7 : 1.0;

    // 7. 最终伤害
    let damage = base * skill.multiplier * critMult * defReduction * attackDownMult * gradeMult * dimensionMult;

    // 8. 次元秒杀机制（神话 vs 凡人）
    if (dimensionMult >= 3.0) {
        const minOneShotDamage = Math.floor(defender.stats.hp * 0.8);
        damage = Math.max(damage, minOneShotDamage);
    }

    damage = Math.max(1, Math.round(damage));

    // 9. 秒杀判定
    const isOneShot = damage >= defender.stats.hp * 0.6;

    return { damage, isCrit, isOneShot, gradeMult, dimensionMult };
}
```

### 9.4 关键公式详解

#### 暴击率
```
crit_rate = min(0.35, (speed + intelligence) / 400)
```
**含义**：速度 + 智力决定暴击率，速度型与智力型角色有暴击优势。
**上限**：35%

#### 防御减免
```
def_reduction = 1 - (defense / (defense + 200))
```
**含义**：防御越高，减免越接近 1（即免伤 100%）。
**曲线**：
- defense=0 → 减免 0%
- defense=100 → 减免 33%
- defense=200 → 减免 50%
- defense=400 → 减免 67%
- defense=800 → 减免 80%

#### 等级压制（[battle-engine.js:30-39](file:///e:/万卷浮生/万卷浮生_Web应用/battle-engine.js#L30-L39)）

```javascript
function getGradeMult(attacker, defender) {
    const diff = GRADE_VALUE[attacker.grade] - GRADE_VALUE[defender.grade];
    if (diff >= 2) return 2.0;   // S打B：2倍
    if (diff === 1) return 1.5;  // S打A 或 A打B：1.5倍
    if (diff <= -2) return 0.5;  // B打S：减半
    if (diff === -1) return 0.8; // A打S 或 B打A：8折
    return 1.0;                  // 同级
}
```

**设计意图**：
- 同级内平衡（1.0 倍）
- 跨级保留原作差距（2.0 倍 / 0.5 倍）
- 一级差距温和（1.5 倍 / 0.8 倍）

#### 次元压制（[battle-engine.js:18-27](file:///e:/万卷浮生/万卷浮生_Web应用/battle-engine.js#L18-L27)）

```javascript
const MYTH_SOURCES = ['西游记', '封神演义', '山海经', '白蛇传', '奥特曼'];

function getDimensionMult(attacker, defender) {
    const atkMythic = MYTH_SOURCES.includes(attacker.source);
    const defMythic = MYTH_SOURCES.includes(defender.source);
    if (atkMythic && !defMythic) return 3.0;  // 神话打凡人：3倍
    if (!atkMythic && defMythic) return 0.5;  // 凡人打神话：0.5倍
    return 1.0;
}
```

**效果**：
- 神话打凡人：3 倍伤害（接近秒杀）
- 凡人打神话：0.5 倍伤害（几乎无效）
- 同类对决：1.0 倍

**设计意图**：
- 体现"金箍棒一万三千五百斤"对凡人躯体的毁灭性
- 凡人打神话几乎无效，体现"人力难敌神力"

### 9.5 技能选择 AI（[battle-engine.js:124-156](file:///e:/万卷浮生/万卷浮生_Web应用/battle-engine.js#L124-L156)）

```javascript
function selectSkill(character, currentHp, currentMp) {
    const hpRatio = currentHp / character.stats.hp;
    const availableSkills = character.skills.filter(s =>
        s.type !== 'passive' && currentMp >= s.mp_cost
    );

    // MP 不足 → 普攻
    if (availableSkills.length === 0) {
        return character.skills.find(s => s.mp_cost === 0 && s.type !== 'passive');
    }

    // 血量 < 30% → 优先防御/回血
    if (hpRatio < 0.3) {
        const defensive = availableSkills.find(s => s.type === 'transform');
        if (defensive && Math.random() < 0.5) return defensive;
        const healSkill = availableSkills.find(s => s.type === 'heal');
        if (healSkill && Math.random() < 0.6) return healSkill;
    }

    // MP 充足 → 70% 选最高倍率，30% 随机
    const attackSkills = availableSkills.filter(s =>
        ['physical_attack', 'magic_attack', 'speed_attack', 'summon'].includes(s.type)
    );
    if (attackSkills.length > 0) {
        attackSkills.sort((a, b) => b.multiplier - a.multiplier);
        if (Math.random() < 0.7) return attackSkills[0];
        return attackSkills[Math.floor(Math.random() * attackSkills.length)];
    }

    return availableSkills[0];
}
```

**AI 决策树**：
1. MP 是否够放技能？→ 普攻
2. HP < 30%？→ 50% 防御，60% 回血
3. MP 充足？→ 70% 选最高伤害技能，30% 随机

- **虚拟列表**：角色 > 100 时启用

```javascript
// 推荐：express-rate-limit
const rateLimit = require('express-rate-limit');
```

```javascript
// 当前
```

```javascript
// 每日限额
const dailyLimits = new Map(); // userId → {date, count}

function checkLimit(userId, maxPerDay = 3) {
    const today = new Date().toISOString().slice(0, 10);
    const userLimit = dailyLimits.get(userId) || {date: today, count: 0};

    if (userLimit.date !== today) {
        userLimit.date = today;
        userLimit.count = 0;
    }

    if (userLimit.count >= maxPerDay) return false;

    userLimit.count++;
    dailyLimits.set(userId, userLimit);
    return true;
}
```
