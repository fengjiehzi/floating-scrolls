# AI 协作规则

在本项目中创作、改写或审查小说时，必须遵守以下规则。

## 开始前

1. 先读 `00_项目管理/项目状态.md`。
2. 再读 `01_小说圣经/小说圣经.md`、`02_故事规划/故事总纲.md` 和 `02_故事规划/连续性台账.md`。
3. 写章节前必须读取该章 `任务卡.md`；没有任务卡就不写正文。
4. 只把已定稿正文视为既成事实，草稿中的新设定不能自动写入小说圣经或台账。

## 创作中

- 不擅自新增会改变主线的势力、能力、规则、身份或巧合。
- 不替角色知道其尚未获得的信息。
- 不为追求文采牺牲人物动机、因果或场景目标。
- 本章必须发生不可撤销的变化；若章末状态与章初相同，视为无效章节。
- 发现资料冲突时标记 `【待确认：冲突说明】`，不得自行选择一个版本。
- 任务卡未要求的支线只能点到为止，不能夺走本章焦点。

## 完成后

1. 先依据 `05_质量控制/检查清单.md` 自检。
2. 审稿结论写入本章 `审稿.md`，问题必须引用具体段落或事件。
3. 只有审稿通过后，才更新连续性台账和项目状态。
4. 修改小说圣经或总纲时，在 `00_项目管理/项目状态.md` 的变更记录中说明原因和影响章节。
