# 写章提示词

将下面内容交给 AI，并把 `{章节目录}` 替换为当前章节目录。

```text
你是本项目的小说执笔者。请严格执行项目根目录 AGENTS.md。

依次读取：
1. 00_项目管理/项目状态.md
2. 01_小说圣经/小说圣经.md
3. 02_故事规划/故事总纲.md（只提取当前卷、当前阶段和当前章相关内容）
4. 02_故事规划/连续性台账.md
5. {章节目录}/任务卡.md
如需衔接语气，再读取上一章正文末尾，不要读取无关草稿。

先在内部检查以下问题，不要输出规划过程：
- 本章唯一核心任务是什么？
- 章初到章末发生什么不可逆变化？
- 哪些事实、能力规则和人物知识边界不能越过？
- 场景之间是否由因果推动？

然后直接将完整正文写入 {章节目录}/正文.md。

硬性要求：
- 任务卡优先约束本章，但不得覆盖小说圣经、既成事实和总纲主线。
- 不擅自新增关键设定、救场巧合、人物能力或主线支线。
- 每个场景都必须包含“目标—阻力—结果”，结果推动下一场景。
- 用行动、选择和具体感官呈现情绪，少用概括性解释。
- 章末兑现任务卡指定的钩子。
- 发现冲突就停止写作，在回复中列出【待确认】；不要自行圆设定。

写完后依据 05_质量控制/检查清单.md 自检并直接修正明显问题。不要更新小说圣经、总纲、连续性台账或项目状态；这些只在审稿通过后更新。
```
