# 0001: 采用 React + Vite + Zustand + Tailwind CSS 技术栈

## 状态

已采纳

## 上下文

「万卷浮生」项目原有前端为纯原生 HTML+CSS+JS 单体架构，`app.js` 包含了大量业务逻辑，代码耦合度高，难以维护和扩展。随着项目功能的增长（角色卡牌战斗、剧情生成、AI配置管理等），需要进行全量UI重构。

重构目标包括：
1. 视觉风格升级（奇幻华丽+暗黑神秘）
2. 架构级重构（组件化体系）
3. 功能+视觉双重构（交互模式升级、响应式优化）

## 决策

采用 **React + Vite + Zustand + React Router + Tailwind CSS** 技术栈进行前端全量重构。

## 备选方案

### 方案 A：保持原生 JS
- **优点**：零依赖、部署简单、学习成本低
- **缺点**：开发效率低、代码难以组织、状态管理混乱、难以实现复杂动画

### 方案 B：引入轻量库（无框架）
- **优点**：平衡开发效率和复杂度
- **缺点**：缺乏组件化体系、状态管理不够成熟、社区生态有限

### 方案 C：Vue 3 技术栈
- **优点**：学习曲线平缓、模板语法直观、Pinia 状态管理成熟
- **缺点**：生态不如 React 丰富、动画库选择较少

### 方案 D：React 技术栈（选中）
- **优点**：生态最丰富、社区活跃、函数式组件和 Hooks 适合复杂交互、Zustand 轻量高效、动画库（GSAP/Framer Motion）成熟
- **缺点**：学习曲线较陡、迁移成本较高

## 理由

1. **复杂交互需求**：游戏类产品需要处理复杂的战斗动画、技能特效、实时状态更新，React 的函数式组件和 Hooks 模型更适合处理这些场景

2. **动画效果需求**：项目追求"奇幻华丽+暗黑神秘"的视觉风格，需要强大的动画能力。React 生态中的 GSAP 和 Framer Motion 是业界最成熟的动画解决方案

3. **状态管理需求**：战斗状态、角色数据、AI配置等需要全局共享，Zustand 提供了轻量高效的状态管理方案

4. **社区生态**：React 拥有最丰富的社区资源和第三方库，遇到问题时更容易找到解决方案

5. **开发效率**：Vite 提供了极速的开发体验，Tailwind CSS 大幅提升样式开发效率

## 影响

### 正面影响
- 开发效率显著提升
- 代码结构清晰，易于维护和扩展
- 动画效果更加丰富和流畅
- 响应式设计更加便捷

### 负面影响
- 迁移成本较高，需要重写大部分前端代码
- 学习曲线较陡，团队需要适应新的技术栈
- 构建产物体积可能增加
- 需要配置构建流程和开发工具
